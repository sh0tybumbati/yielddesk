# Research Plan

## Priority 1 — Engineer pain points
Search forums (Reddit r/chipdesign, r/VLSI, EEVblog, Hacker News, Stack Exchange EE) for:
- "yield calculator" complaints or requests
- Questions about D₀, defect density, Murphy's Law yield
- What data engineers have to look up manually

## Priority 2 — Competitive landscape
Find and document every existing yield calculator tool:
- Web tools, Excel sheets, SPICE models, academic tools
- What each does well/poorly
- Gaps that nothing fills

## Priority 3 — Data sourcing
- Public D₀ data from ITRS/IRDS roadmaps
- Wafer cost data beyond current YieldDesk values
- Mask cost data at advanced nodes
- Panel/interposer cost data

## Priority 4 — Feature ideas
- Panel/chiplet/2.5D calculations
- What visualizations engineers would find most useful
- SEO keywords engineers actually use
